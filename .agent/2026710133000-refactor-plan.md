# Plan: 改名 Nota + 六边形架构 + 工作区模式

## 目标
1. 项目改名 Nota（README/文档标题；crate `nota-core` 名称保留为"核心模块"）。
2. 改造为 Cargo workspace，含三 crate：`nota-core`（domain + ports）、`nota-infra`（adapter 集合）、`nota-cli`（binary，接线 + 日志初始化）。
3. 六边形架构：nota-core 完整领域纯净 + 仓储端口；core 无全局状态（全 DI）；core 日志只用 `log` 门面。

## 已确认决策
- **领域纯净**：`Metadata`/`Message`/`Schedule`/`Session` 不再 derive `crudly::*`/`sqlx::FromRow`；`Session` 不再持有 `SqlitePool`。持久层映射在 infra 完成。
- **工作区**：`nota-core` + `nota-infra`（adapter 可些许耦合但尽量少）+ `nota-cli`。
- **全 DI**：移除 `OnceLock<SessionManager>`/`OnceLock<PersonaManager>`/`static BASE_DIR`；core 通过构造注入 port（`Arc<dyn ...>`），由 nota-cli 在 main 组装。
- **日志边界**：`log` 仅指日志门面；core 仍可用 `serde`/`async-trait`/`chrono`/`anyhow`。core 内 `tracing::*` 全部替换为 `log::*`。nota-cli 用 `tracing-subscriber` + `tracing-log` 桥接把 `log` 路由进 tracing；cli 自身代码仍可直接用 `tracing`。

## core port 定义（nota-core）
- `SessionHandler`（保留现有 trait）：`async fn handle(&self, session_id: &str, messages: &[Message]) -> Result<()>`。
- `LlmClient`（保留）：`async fn chat(&self, system: &str, messages: &[Message]) -> Result<String>`。
- `SessionRepository`（新增 port）：`create_session`、`load_all`（返回元数据/消息/调度，用于冷启动装填）、`insert_message`、`set_archive_at`、`archive_session`。返回 `anyhow::Result`。pool/迁移/move-to-archive 均为 infra 实现细节。（`list_metadata` 不在 port 上：冷启动 `load_all` 后内存 HashMap 是权威缓存，列表查询由 `SessionManager` 直接读内存，避免重复查库。）
- `PersonaStore`（新增 port）：`read_persona_file(name, filename)`、`create_persona(name)`。mtime 缓存为 infra 内部细节。

## core 服务（DI，无全局状态）
- `SessionManager`：持 `Arc<dyn SessionRepository>` + in-memory `RwLock<HashMap<String, Session>>`（Session 不含 SqlitePool）。`new_session`/`insert_message`/`set_archive_at`/`list_metadata`/`archive_expired_sessions` 经 repo 持久化 + 更新内存。handler 通过 `register_handler(sid, Arc<dyn SessionHandler>)` 注入（用 `Arc` 而非 `Box`，以便 `register_handler_all` 共享同一 handler 给多个 session）。
- `PersonaManager`（合并旧 PersonaManager+PersonaHandler）：持 `Arc<dyn PersonaStore>` + `Arc<dyn LlmClient>` + `RwLock<Option<Persona>>`，并 `impl SessionHandler`（handle 内读 persona 文件→拼 system→llm.chat）。移除 `PersonaHandler::new()` 与 `PersonaManager::get()`。

## infra adapter 映射
- `SqliteSessionRepository`：吸收 `Session::load_from`/`Session::new`（`sqlx::migrate!`、`crudly::InsertNoId/InsertWithoutId`/`SelectAll`）；内部维护 `session_id -> SqlitePool(max_connections=1)`；archive 用 `fs::rename`。row 结构（带 crudly/sqlx derives）仅存于此 crate，与 core 实体一一映射。
- `FilePersonaStore`：吸收 `Persona::read_file`（mtime 缓存）与 `Persona::create`（写 `assets/solo.md` 模板 + 空 `memory.md`）。
- `StubLlm`：原样迁入。
- `http`：吸收 `connect/`（axum router + handler），改为从构造注入的 `Arc<SessionManager>` 取数据（不再 `SessionManager::get()`）。
- `config`：吸收 `config/mod.rs`（Config + load/save），wizard 拆到 cli。
- 路径：`base_dir()` 解析移到 cli（`dirs::home_dir().join(".nota")`），注入给各 adapter 构造函数；core 不感知路径。

## CLI（nota-cli/main.rs）
- 保留 `ensure_dir`（用注入的 base_dir）。
- `init_tracing` 保留（控制台 + 30 天轮转文件层），新增 `tracing_log::LogTracer::init()` 桥接 core 的 `log::*`。
- 装配：建 `SqliteSessionRepository`、`FilePersonaStore`、`StubLlm` → 注入 `PersonaManager` → 把 `PersonaManager`（作为 `SessionHandler`）注册进 `SessionManager` → 启动 http。冷启动：`SessionManager::load_all` 经 repo 装填 + 注入 handler。
- 优雅关停（`CancellationToken`）保留。

## 风险/注意
- crudly 0.6 锁 sqlx 0.9，版本不可擅动（见 guide.md）。
- `sqlx::migrate!` 宏路径相对编译它的 crate；迁移文件必须落在 `nota-infra` 内并更新宏路径，否则编译期找不到。
- `Session` 去掉 `SqlitePool` 后，`insert_message` 的"插入后 push 到内存 + 触发 handler"逻辑要保留（含其错误处理：handler 出错只 `log::error!` 不阻断）。
- 全 DI 后 `SessionManager` 不再 `Send + Sync` 全局可 `::get()`；http handler 需 `Arc<SessionManager>` clone 注入到 Router state，注意 axum 0.8 state 用法。
- 根目录文件夹仍为 `nota-core`、GitHub 仓库为 `Nota.Core`；"改名 Nota" 仅落地于文档/README。仓库/文件夹重命名为用户侧手动操作，**不在代码改动范围**（列出供知会）。
